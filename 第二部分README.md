# SmartCinema 智能影院选座系统 — 模块1 & 2 实现说明

## 1. 模块概览

本部分在基础框架（登录注册 + Canvas 座位图）之上，实现了 **智能推荐选座（模块1）** 和 **手动选座增强（模块2）**，完善了影院选座系统的核心交互流程。两个模块集成于 `cinema.html` 页面中，通过右侧边栏切换使用。

---

## 2. 模块1：智能推荐选座

### 2.1 功能描述

用户在右侧边栏「智能推荐」标签页中，选择票型、输入人数、填写每位成员的姓名和年龄，点击「推荐座位」按钮后，系统根据预设规则自动计算最佳区域和座位，并在 Canvas 座位图上以 **蓝色高亮（带闪烁边框）** 显示推荐结果。

### 2.2 支持的票型与输入

| 票型 | 人数范围 | 成员信息 | 推荐偏好 |
|------|----------|----------|----------|
| **个人票** | ≥1 人 | 姓名 + 年龄 | 尽量居中，兼顾中后排 |
| **情侣票** | 固定 2 人 | 姓名 + 年龄 | 优先中间区域连续双座 |
| **家庭票** | ≥2 人 | 姓名 + 年龄 | 优先中后排连续座位 |
| **团体票** | 5~20 人 | 姓名 + 年龄 | 同一排连续座位，优先中间排 |

- 所有票型均支持动态增减成员（先输入人数，自动生成对应数量的姓名/年龄输入框）。
- 年龄用于判断少年（<15岁）或老年（≥60岁），并作为硬约束依据。
- 团体票人数超出范围（5~20）时，前端给出明确错误提示。

### 2.3 排座规则

#### 硬约束（年龄避让，强制执行）

- **少年（＜15岁）**：禁止坐在前三排（第1~3排，索引0~2）。
- **老年（≥60岁）**：禁止坐在最后三排（倒数第1~3排）。
- 若团体/家庭中同时包含少年和老年，则取并集约束，即座位行必须同时避开前3排和后3排，只能坐中间区域。

#### 软偏好（评分排序，择优推荐）

1. 筛选出满足硬约束的所有行。
2. 在每行中扫描 **连续空座（绿色）** 段，选出长度 ≥ 所需人数的候选段。
3. 对每个候选段计算评分（分数越高越优），评分规则如下：
   - **情侣**：中心距离越小评分越高，连续段长度为偶数时额外加分。
   - **家庭**：靠近中后排（约总行数×0.6）加分，中心距离小加分。
   - **团体**：靠近中间排（总行数×0.5）加分，中心距离小加分。
   - **个人**：兼顾中间排和中心距离，综合评分。
4. 选取得分最高的候选段，将其中的座位标记为「推荐」（蓝色），并在结果框中显示具体排号和座位号。

若不存在任何满足硬约束且长度足够的连续空位，系统会提示错误，不进行高亮。

### 2.4 交互方式

- 点击「推荐座位」按钮后，算法会 **清除上一次的推荐高亮**，重新计算并显示新的推荐座位。
- 推荐座位（蓝色）**可以像空座一样被点击或拖拽**，一旦操作即变为「选中」（黄色），同时自动清除所有蓝色推荐高亮（表示用户已接受推荐或自行调整）。
- 再次点击推荐按钮，会基于当前座位状态（空座和已售）重新计算，不会保留之前的蓝色标记。

## 3. 模块2：手动选座增强

### 3.1 功能描述

在「手动选座」模式下，用户可直接在 Canvas 座位图上进行 **单选**、**多选（Ctrl+单击）** 和 **拖拽滑动选座**，自由选择心仪座位。所有手动操作均与推荐结果互操作。

### 3.2 操作方式

| 操作 | 行为 | 状态变化 |
|------|------|----------|
| **单击座位** | 切换状态 | 空座/推荐 → 选中（黄）；选中 → 空座（绿）；已售不变 |
| **Ctrl + 单击座位** | 多选切换 | 与单击逻辑相同，便于同时切换多个座位 |
| **拖拽滑动（按住左键滑动）** | 画笔连续切换 | 滑过的座位按「空座→选中，选中→空座」切换，已售跳过；可快速连续选中/取消多个座位 |
| **清空所有选中** | 按钮（侧边栏） | 将所有黄色座位恢复为空座，同时清除推荐高亮 |

- 拖拽选座在桌面端使用鼠标实现，移动端虽无触摸拖拽，但可通过单击完成所有功能。
- 所有操作后，座位状态**立即保存至 LocalStorage**（但黄色状态在保存时会被映射为空座，因此刷新页面后黄色消失，而红色已售保留）。

### 3.3 与推荐座位的交互

- 当推荐座位高亮（蓝色）出现后，用户可以通过 **单击或拖拽** 将任意蓝色座位变为黄色（选中），此时系统自动清除所有剩余蓝色高亮，并隐藏推荐结果框。
- 若用户对推荐结果不满意，也可直接手动点击其他空座进行选择，推荐高亮不会自动消失，但一旦操作任何座位（包括点击或拖拽），都会清除推荐高亮，避免干扰。

## 4. 数据持久化与状态管理

- **用户数据**：`localStorage` 键 `smartCinemaUsers`，存储所有注册用户信息。
- **座位状态**：`localStorage` 键 `smartCinemaSeats`，按影厅（small/medium/large）分别存储二维状态数组。
- **持久化规则**：
  - `EMPTY`（绿色）和 `SOLD`（红色）状态 **永久保存**。
  - `SELECTED`（黄色）和 `RECOMMENDED`（蓝色）状态 **不保存**：保存时自动映射为 `EMPTY`。
- **设计意图**：选中（黄）仅是临时操作，刷新页面后应重置为空座；已售（红）为真实销售数据，必须持久化。

---

## 5. 交互流程示例

1. **登录** → 进入 `cinema.html`，默认显示「智能推荐」面板。
2. **推荐**：选择「家庭票」，人数 3，年龄分别为 40、38、12 岁（少年），点击推荐 → 系统自动避开前三排，在中后排推荐连续 3 个空座，蓝色高亮。
3. **手动调整**：用户觉得推荐位置偏左，切换到「手动选座」，单击偏右的座位选中，或按住 Ctrl 多选，蓝色高亮自动清除。
4. **拖拽选座**：按住鼠标左键在座位区滑动，快速选中一排座位。
5. **切换影厅**：点击「中厅」，所有座位状态重新加载（黄色清空），推荐高亮消失。
6. **刷新页面**：所有黄色消失，红色已售状态保留，空座保持。

## 6. 技术实现要点

- **状态扩展**：新增 `RECOMMENDED = 3` 状态，并在 `COLORS` 中定义蓝色。
- **闪烁边框**：推荐座位采用 `setInterval` 定时切换边框颜色，营造呼吸效果，提示用户。
- **推荐算法**：采用「硬过滤 → 候选段扫描 → 评分排序」三步策略，兼顾约束和偏好。
- **拖拽选座**：监听 `mousedown`/`mousemove`/`mouseup`，使用 `lastDraggedSeat` 防止同一座位重复切换，提升流畅度。
- **存储映射**：在 `saveSeatsToLocal` 中将 `SELECTED` 和 `RECOMMENDED` 统一转为 `EMPTY`，实现黄色不持久化。

---

## 7. 注意事项

- 推荐算法基于 **当前座位状态**（空座和已售）计算，黄色座位（临时选中）不会被纳入推荐（因为保存时会被视为空座，但运行时它们仍为黄色，算法只扫描 `EMPTY`）。
- 团体票必须同排连续，若人数超出某排空段长度，算法会尝试其他排，若所有排均不满足则报错。
- 年龄约束同时作用于所有成员，团体中若既有少年又有老年，则必须避开前3排和后3排，只能坐中间区域。
