# SmartCinema 智能影院选座系统 — 模块3 & 4 实现说明

## 1. 模块概览

本部分在基础框架及模块1、2之上，实现了：

- **模块3：影院热度地图**——在原有 Canvas 影院座位图中展示红、黄、蓝三档热度，并支持一周七天动态切换和自动播放。
- **模块4：观影体验评分**——根据视角、距银幕距离、周围空座三个指标计算系统评分；观众可在观影后提交1～5星评价，系统生成综合结果。

两个模块均集成在 `cinema.html` 右侧功能栏中，不依赖第三方图表库或后端服务。

---

## 2. 模块3：影院热度地图

### 2.1 功能描述

用户点击右侧「热度地图」标签后，原 Canvas 座位图切换为热度分析视图：

- **红色**：热门座位，热度70～100；
- **黄色**：一般座位，热度45～69；
- **蓝色**：冷门座位，热度0～44。

座位颜色和周围的半透明光晕都由 Canvas 原生 API 绘制。鼠标悬停时会显示座位号、热度数值和分类。

### 2.2 一周动态变化

侧边栏提供星期一至星期日滑块和「播放一周变化」按钮：

1. 拖动滑块可以直接查看某一天的热度；
2. 点击播放按钮后，系统每1.1秒切换一天；
3. 离开热度地图标签时自动停止播放，避免后台定时器继续运行；
4. 最近选择的星期保存在 `localStorage` 的 `smartCinemaHeatmapDay` 中。

### 2.3 热度算法

每个座位的热度由四部分组成：

1. **排距偏好**：以中后排为主要热门区，不同日期的理想排位有轻微变化；
2. **中心偏好**：越接近银幕中轴，热度越高；
3. **星期修正**：周末整体需求高于工作日；
4. **稳定扰动**：使用座位行、列和星期生成固定伪随机值，使同一区域内也有自然差异，同时保证同一天重复查看结果一致。

实现中使用高斯衰减函数模拟“离理想位置越远，热度越低”的规律，再将结果限制在0～100范围。所有计算均在浏览器本地完成。

### 2.4 Canvas 绘制方式

- `drawHeatmapLayer()` 为每个座位绘制径向渐变光晕；
- `drawSeat()` 在热度模式下改用热度色板；
- `globalCompositeOperation = "screen"` 使相邻光晕自然叠加；
- 影厅切换后会基于新影厅座位数重新统计热门、一般和冷门座位数量。

## 3. 模块4：观影体验评分

### 3.1 使用流程

1. 在「智能推荐」或「手动选座」中选中一个或多个座位；
2. 切换到「体验评分」，查看系统分、等级和三项指标；
3. 观影后选择1～5星并提交，查看综合结果。

未选座时评分面板不会给出无意义分数，而是提示用户先完成选座。

### 3.2 系统评分指标

| 指标 | 权重 | 计算思路 |
|---|---:|---|
| 视角舒适度 | 40% | 座位越靠近银幕中轴得分越高，最侧边座位得分降低 |
| 银幕距离 | 35% | 以总排数约58%的位置作为舒适距离，过近或过远均扣分 |
| 周围空座 | 25% | 统计座位周围最多8个邻位，空座比例越高得分越高 |

多选座位时，先分别计算每个座位的三项指标，再取平均值生成整组座位的系统分。

### 3.3 等级划分

| 分数范围 | 结果 |
|---|---|
| 85～100 | 极佳 |
| 70～84 | 优秀 |
| 0～69 | 一般 |

系统同时根据三项指标生成自然语言分析，说明视角、排距和周围空间的优缺点，帮助用户理解评分逻辑。

### 3.4 观众评分与综合结果

观众可选择1～5星，星级先换算为百分制，再与系统分合成：

```text
综合分 = 系统分 × 70% + 观众分 × 30%
观众分 = 星级 × 20
```

系统分占较高权重，用于保持座位客观条件的稳定性；观众分反映实际观影感受。综合分仍按「极佳、优秀、一般」三档输出。

### 3.5 数据持久化

评分记录保存在 `localStorage` 的 `smartCinemaExperienceRatings` 中，以“用户 + 影厅 + 座位组合”为键，内容包括：

- 观众星级和百分制分数；
- 系统分、综合分和等级；
- 影厅、座位列表；
- 保存时间。

用户再次选择同一组座位时，系统会自动恢复此前保存的评分结果。

## 4. 与原有模块的协作关系

- 模块3复用模块1、2的 `seats` 二维数组和 Canvas 布局坐标，只改变分析视图，不修改座位状态。
- 热度模式下禁止点击或拖拽选座，避免用户把热度颜色误认为座位状态。
- 模块4直接读取模块1推荐后确认的黄色座位，或模块2手动选择的黄色座位。
- 切换影厅时，系统重新计算热度统计并清空当前临时选座评分面板。
- 已售、选中、推荐状态仍沿用原有 `smartCinemaSeats` 持久化规则。

---

## 5. 响应式与交互设计

- 右侧功能栏扩展为四个按钮组成的2×2网格，避免窄屏文字拥挤。
- PC端使用左右布局；屏幕宽度小于860px时，座位图与功能面板改为上下布局。
- 热度统计使用三列信息卡；评分使用圆环总分、进度条指标和星级按钮。
- 所有核心流程控制在3步以内，符合“Don't Make Me Think”设计原则。
