# SmartCinema 智能影院选座系统 — 基础框架说明

## 1. 项目简介

本项目是清华大学前端小学期大作业「SmartCinema 智能影院选座系统」的基础实现。
目前已完成大作业要求中的「基本功能设计与实现」部分（30分），包括：

- (1) 登录注册功能（10分）
- (2) 主界面 Canvas 座位图（20分）

后续 6 个模块（智能推荐、手动选座、热度地图、观影评分、无障碍模式、订单中心）
可在此基础上继续扩展。

## 2. 文件结构

| 文件 | 说明 |
|------|------|
| `login.html` | 登录注册入口页面。登录/注册 Tab 切换、普通用户注册、默认管理员账号登录、登录成功后跳转 `cinema.html` |
| `cinema.html` | 影院选座主界面。Canvas 绘制弧形座位图、小/中/大厅三厅切换、座位状态（空座/选中/已售）、点击选座交互、显示当前登录用户信息 |
| `README.md` | 本文档，说明基础框架的实现思路与后续扩展提示 |

## 3. 登录注册实现思路

### 3.1 无后端、纯前端存储

项目不使用后端数据库，用户信息存放在浏览器的 `localStorage` 中。

- 存储键名：`smartCinemaUsers`
- 数据结构：

```json
{
    "用户名": {
        "password": "密码",
        "role": "member" | "admin",
        "registerTime": "2026-07-20T13:45:51.000Z"
    }
}
```

### 3.2 默认管理员

系统初始化时（`login.html` 加载时），`initDefaultAdmin()` 会检查 `localStorage` 中是否已存在名为 `admin` 的用户。如果不存在，则自动创建：

> **用户名**：`admin`
> **密码**：`admin123`
> **角色**：`admin`

这是系统唯一的管理员账号，通过登录页直接输入 `admin / admin123` 即可登录。

### 3.3 普通用户注册

注册页面只收集用户名、密码、确认密码，不提供角色选择。所有新注册用户的 `role` 强制写死为 `"member"`（普通用户/会员）。

这意味着：

- 任何人都不能通过注册自行成为管理员
- 管理员只能由系统预设
- 符合真实系统中「角色不能由用户自己声明」的安全原则

### 3.4 登录验证流程

1. 读取用户名和密码输入
2. 从 `localStorage` 中读取 `smartCinemaUsers`
3. 检查用户名是否存在
4. 检查密码是否匹配
5. 登录成功后将会话写入 `sessionStorage`：
   - 键名：`smartCinemaCurrentUser`
   - 内容：`{ username, role, loginTime }`
6. 跳转至 `cinema.html?user=xxx&role=xxx`

### 3.5 会话管理

- `localStorage`：持久化存储，关闭浏览器后用户数据仍存在
- `sessionStorage`：会话级存储，关闭标签页/浏览器后自动清除，用于保存当前登录状态

### 3.6 登录状态在 cinema.html 中的使用

`cinema.html` 通过读取 URL 参数和 `sessionStorage` 判断当前登录用户，在页面顶栏显示用户名和角色标签：

- 普通用户：绿色「会员」标签
- 管理员：红色「管理员」标签

点击「登出」按钮会清除 `sessionStorage` 并返回 `login.html`。

## 4. 管理员账号登录方式

打开 `login.html`，在登录表单中输入：

> **用户名**：`admin`
> **密码**：`admin123`

点击「登录」即可进入 `cinema.html`，此时顶栏会显示「管理员」标签。

> **提示**：`admin` 账号的数据保存在 `localStorage` 中。如果曾经注册过同名用户，可能会与默认管理员冲突。清除浏览器 `localStorage` 或更换浏览器即可恢复默认。

## 5. Canvas 座位图实现思路

### 5.1 三个放映厅规格

| 放映厅 | 排数 | 每排座位 | 总座位 |
|--------|------|----------|--------|
| 小厅 | 10 | 10 | 100 |
| 中厅 | 10 | 20 | 200 |
| 大厅 | 10 | 30 | 300 |

### 5.2 弧形排列计算

座位不是简单的矩形网格，而是沿圆弧扇形展开：

- 银幕位于画布顶部，对应圆心位置
- 第 1 排半径最小，离银幕最近，座位最密集
- 第 10 排半径最大，离银幕最远，座位最稀疏
- 左右对称分布，中间正对银幕

单个座位坐标使用极坐标转换得到：

```
radius = 基础半径 + 排号 × 排间距
angle  = -总跨度/2 + (列号 / (每排座位数-1)) × 总跨度
x = 圆心 x + radius × sin(angle)
y = 圆心 y + radius × cos(angle)
```

不同厅的弧形总跨度不同，座位越多跨度越大，以保证排列自然。

### 5.3 座位状态与颜色

| 状态 | 颜色 | 含义 |
|------|------|------|
| `empty` | 绿色 | 可选 |
| `selected` | 黄色 | 用户已选中但未付款 |
| `sold` | 红色 | 已被购买 |

每个座位对象保存状态字段：`status = 'empty' | 'selected' | 'sold'`，绘制时根据 `status` 设置 `ctx.fillStyle`。

### 5.4 交互

- 点击空座 → 变为选中（黄）
- 再次点击选中座位 → 取消选中，变回空座（绿）
- 已售座位（红）不可点击
- 鼠标悬停时显示提示框，展示「X排X座 — 状态」

### 5.5 响应式

Canvas 使用 `devicePixelRatio` 进行高清渲染，尺寸随窗口 `resize` 自动重新计算，适配 PC、iPad、手机等不同屏幕。

## 6. 数据存储汇总

| 用途 | 存储方式 | 键名 |
|------|----------|------|
| 所有用户信息 | `localStorage` | `smartCinemaUsers` |
| 当前登录会话 | `sessionStorage` | `smartCinemaCurrentUser` |
| 座位状态（待扩展） | 建议 `localStorage` | 例如 `smartCinemaSeats` |
| 订单记录（待扩展） | 建议 `localStorage` | 例如 `smartCinemaOrders` |

## 7. 后续模块开发提示

### 7.1 智能推荐选座（模块1）

在 `cinema.html` 中增加推荐算法，根据已选座位或历史数据计算最佳座位，用特殊颜色（如蓝色高亮）在 Canvas 上标出推荐位置。

### 7.2 手动选座（模块2）

已部分实现（点击选座）。可继续扩展为：限制最大选座数、多选连座、显示选座列表、计算总价等。

### 7.3 热度地图（模块3）

在座位图上叠加透明度或颜色深浅，表示座位热度。热度数据可来自历史订单或模拟数据。

### 7.4 观影评分（模块4）

在 `cinema.html` 增加评分弹窗或侧边栏，用户可对影片/座位体验打分，评分数据存入 `localStorage`。

### 7.5 无障碍模式（模块5）

增加一个模式切换开关，改变 Canvas 颜色方案（高对比度、色盲友好等），并放大文字/座位尺寸。

### 7.6 订单中心（模块6）

新增 `order.html` 或弹窗，管理订单的生成、支付、取消。当用户点击「确认购票」时，将选中座位状态从 `selected` 改为 `sold`，并生成订单记录存入 `localStorage`。

## 8. 注意事项

- 本项目为前端教学演示，密码以明文存储于 `localStorage`。实际生产环境应使用后端数据库 + 密码哈希 + HTTPS
- 所有页面直接用浏览器打开即可运行，无需构建工具或服务器
- 清除浏览器 `localStorage` 会丢失所有用户数据和座位状态
